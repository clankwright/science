# Longevity Research Knowledge Base

Scripted Karpathy-style wiki. The human curates sources; the LLM maintains
the wiki (paper pages, topic links, analyses, index).

## Structure

```
sources.json                # canonical 40-source manifest (source of truth)
LICENSES.md                 # generated by scripts/licenses.py
log.md                      # append-only operation log
sources/                    # raw downloads, gitignored
  html/                     # fetched by scripts/download.py
  pdf/                      # fetched by scripts/download.py
  md/                       # converted by scripts/convert.py
wiki/                       # LLM-authored knowledge layer
  index.md                  # master catalog + orientation
  papers/<id>.md            # one page per source in sources.json
  topics/<slug>.md          # one page per research topic
  analysis/<slug>.md        # syntheses, tier rankings, open questions
  build/                    # index artifacts, gitignored
scripts/                    # the pipeline, all idempotent
  download.py
  convert.py
  index.py
  lint.py
  licenses.py
  _wiki.py                  # shared parsing helpers
```

## Wiki article format

Every page starts with YAML front matter. Required fields differ by kind.

**Paper page** (`wiki/papers/<id>.md`):

```markdown
---
id: <slug matching sources.json id>
title: "<source title>"
url: <canonical URL>
year: <int>
venue: "<journal or preprint server>"
access: open | preprint | paywalled
kind: paper
topics: [<topic-slug>, <topic-slug>]
---

# <Title>

One-paragraph summary.

## Key findings
- ...

## Why it matters
...

## Related
- [[topics/<slug>]]
- [[papers/<other-id>]]
```

**Topic page** (`wiki/topics/<slug>.md`):

```markdown
---
id: topic-<slug>
topic: <slug>
---

# <Topic name>

One-paragraph orientation.

## Flagship work
- [[papers/<id>]] — one-line summary

## Adjacent
- [[topics/<slug>]]
```

**Analysis page** (`wiki/analysis/<slug>.md`): free-form synthesis. YAML
front matter minimally `id:` and `title:`.

## Naming conventions

- Paper ids: stable, lowercase, kebab-case, match `sources.json` `id`.
- Topic slugs: stable English names, kebab-case, no dates, no versions.
- No numeric prefixes in filenames.

## Cross-references

- Internal: `[[papers/<id>]]`, `[[topics/<slug>]]`, `[[analysis/<slug>]]`.
  Parsed by `scripts/index.py` into a graph; broken links fail lint.
- External: normal markdown `[text](url)`.

## Workflows

### Ingest (scripted)

1. Append the new source to `sources.json` with a stable `id`, best open-access
   URLs (`pdf_url`, `url`, `alt_url` lists permitted), and `topics:`.
2. Add a `LICENSE_MAP` entry in `scripts/licenses.py`.
3. `python3 scripts/licenses.py all`: fills `license` field, regenerates
   `LICENSES.md` and `biology/LICENSES.md`.
4. `python3 scripts/download.py`: fetches HTML/PDF to `sources/`. Re-runnable,
   skips cached. Needs unrestricted egress to arXiv, bioRxiv, medRxiv, PMC,
   Nature, MDPI, etc.
5. `python3 scripts/convert.py`: cleans HTML/PDF into markdown under
   `sources/md/`. Uses trafilatura for HTML, pdftotext for PDF.
6. Read `sources/md/<id>__*.md`. Write `wiki/papers/<id>.md` with YAML front
   matter, summary, key findings, wikilinks.
7. Link the new paper into every topic page in its `topics:` list.
8. Update `wiki/index.md` if the new paper reshapes the field map.
9. `python3 scripts/index.py && python3 scripts/lint.py`. Fix any errors.
10. Append to `log.md`: `YYYY-MM-DD INGEST: <description>`.

Steps 6, 7, 8 are LLM work. Everything else is scripted.

### Query

1. Load `wiki/build/index.json` or grep `wiki/` for relevant pages.
2. Synthesize the answer citing specific `wiki/papers/*` pages.
3. If the answer is load-bearing for future queries, file it as an
   `analysis/*.md` page.
4. Append to `log.md`: `YYYY-MM-DD QUERY: <question summary>`.

### Lint

1. `python3 scripts/index.py && python3 scripts/lint.py`.
2. Lint checks: broken wikilinks, missing front-matter fields, orphan
   topics, papers with no inbound links, `sources.json` to `papers/` id
   parity, malformed URL fields, missing licenses.
3. Fix issues. Re-run.
4. Append to `log.md`: `YYYY-MM-DD LINT: <summary>`.

## Writing style

- NEVER use em dashes. Use colons, semicolons, commas, or restructure.
- Concise, technically confident. No fluff.
- No "I believe", "perhaps", "it seems", "in order to", "utilize".
- Cross-reference liberally; orphan pages are low-value.
- Every factual claim traces back to a paper page.
- Raw sources in `sources/` are immutable; curation happens in `wiki/`.
- When sources disagree, note the disagreement on the topic page with
  citations to each.
- Prefer evidence-weighted language over superlatives.

## Contribution policy

- This is an open-science repo. No AI/LLM attribution in commit messages
  (see root `CLAUDE.md`).
- `sources.json` is the source of truth for which sources exist. `wiki/`
  is regenerable from `sources.json` plus the raw downloads plus human
  judgement; treat paper pages as reproducible artifacts, not manuscripts.
- `sources/html/`, `sources/pdf/`, `sources/md/`, `wiki/build/` are
  gitignored. Commit converted full-text only if the source's `license`
  permits redistribution and include license + DOI in YAML front matter.
